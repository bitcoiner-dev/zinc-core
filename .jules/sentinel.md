## 2025-04-29 - [Fix panic vulnerability in BIP32 derivation index conversions]
**Vulnerability:** Unvalidated inputs to `ChildNumber::from_hardened_idx` and `ChildNumber::from_normal_idx` combined with `.unwrap()` can cause unexpected panics if values exceed the maximum allowed boundaries (e.g. 2^31 - 1). This is a DoS risk if external data can reach these codepaths.
**Learning:** `bdk_wallet` derivation structures require explicit bounds checking for index values before assignment. Replacing `.unwrap()` with proper Result mapping avoids panics and allows the application to fail gracefully.
**Prevention:** Avoid blindly unwrapping `ChildNumber` creation functions or any cast/conversion functions related to external or potentially unbounded inputs. Ensure all such calls use `.map_err` or propagate the error correctly using `?`.
