## 2024-04-09 - Remove unsafe `unwrap()` from dynamic input conversions
**Vulnerability:** Unvalidated dynamic/user-controlled inputs passed to `bdk_wallet::bitcoin::bip32::ChildNumber::from_hardened_idx` (or normal) and `u8::try_from(rec_id.to_i32())` with `.unwrap()` can cause panics and DoS attacks if values exceed acceptable limits (e.g., indices > 2^31 or invalid ECDSA recovery IDs).
**Learning:** Hardcoded indices in `ChildNumber::from_hardened_idx(86).unwrap()` are safe because the input is statically known to be within bounds. However, when parsing dynamic variables (like `account`, `purpose`, `chain`, `index` in key derivation, or `rec_id` from a signature), using `.unwrap()` exposes the application to runtime panics.
**Prevention:** Always use Result-returning wrappers (like the existing `Self::child_hardened` or `Self::child_normal` helpers) or explicitly map errors (`.map_err()`) when dealing with dynamic or potentially unbounded inputs to gracefully propagate errors instead of crashing.
