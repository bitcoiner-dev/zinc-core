## 2025-05-25 - HashMap with_capacity Memory Optimization
**Learning:** In heavily used loops and builders (e.g., PSBT preparation and ordinal auditing), instantiating HashMaps with `HashMap::new()` causes dynamic memory reallocations during insertion as the collection grows. When the exact number of elements to be inserted is known (e.g., `self.inscriptions.len()`), pre-allocating memory using `HashMap::with_capacity(len)` provides a predictable, measurable performance optimization that avoids these runtime reallocation overheads without sacrificing readability or adding complex logic.
**Action:** Always check loop structures and mapping functions. If the length of the source data structure is known, use `HashMap::with_capacity(len)` instead of `HashMap::new()` to prevent unnecessary allocations. Ensure the correct length is passed (e.g. matching `self.inscriptions.len()` when building the inscriptions map).
