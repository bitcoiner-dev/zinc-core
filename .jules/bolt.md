## 2024-04-24 - Pre-allocating HashMaps
**Learning:** Initializing `HashMap` via `HashMap::new()` causes dynamic memory reallocations inside large loop iterations. Since we often iterate over `self.inscriptions` (or similar lists of known size), this is inefficient.
**Action:** Replace `HashMap::new()` with `HashMap::with_capacity(len)` when iterating over collections of known size (like `self.inscriptions.len()` or `w.list_unspent().count()`) to reduce allocations. Use `HashMap::with_capacity_and_hasher(len, Default::default())` when the `HashMap` is generic over a specific Hasher `S`. But note the performance anti-pattern: avoid calling `.count()` on iterators (e.g., `list_unspent().count()`) solely to pre-allocate capacity since it consumes the iterator. Instead, only use `with_capacity` if the length is already known (like from a `Vec` or `slice`).
